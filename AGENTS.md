# GoFlow — הוראות עבודה לסוכנים

## טעינה מחייבת

בכל משימה שעוסקת באחד מהנושאים הבאים חובה לקרוא תחילה את הקובץ:

`skills/goflow-social-campaigns/SKILL.md`

הנושאים המחייבים טעינה:

- קמפיין אורגני או ממומן ברשתות חברתיות.
- WhatsApp Status, Facebook Story, Facebook Post או LinkedIn Post.
- דף נחיתה, אבחון, טופס לידים או CTA.
- קישור קצר, UTM, `click_id` או מדידת משפך.
- Formspree, Google Sheets, לשונית `לידים` או לשונית `קליקים על קישורים`.
- יצירת תמונות, נוסחי סטטוס, פוסטים או חומרי פרסום ל־GoFlow.

אין להסתמך על זיכרון משיחה קודמת במקום לקרוא את ה־Skill ולסרוק את הקוד העדכני ב־`main`.

## סדר העבודה המחייב

1. לקרוא את ה־Skill.
2. לבדוק בפועל את `main`, את דף היעד, סקריפטי המדידה, הטפסים, ה־Workflows והקישורים הקיימים.
3. לא לשאול שאלות שכבר נענו בשיחה או בקבצים.
4. להשלים רק את פרטי הקמפיין החסרים לפי שאלון הקליטה שב־Skill.
5. להפריד בממצאים בין: עובדה, השערה, סיכון והחלטה.
6. לעבוד לפי: מחקר → ממצאים → חלופות → סיכונים → תוכנית → אישור → ביצוע → בדיקות → PR → מיזוג → בדיקת קצה לקצה.
7. אם המשתמש כבר נתן אישור מפורש לביצוע, אין לבקש אישור זהה פעם נוספת; עדיין חובה לבצע את המחקר והבדיקות.

## גבולות שינוי

אין לשנות ללא הצדקה, תוכנית ואישור מפורש:

- Google Ads, GA4 או הגדרות המרה.
- Apps Script או מבנה Google Sheets.
- Formspree endpoint או Workflow של Formspree.
- מנועי אבחון משותפים.
- `assets/shared.js` או מנגנון ייחוס משותף.
- דפי נחיתה וקמפיינים שאינם חלק מהמשימה.
- `main`, Production או FTP ללא ענף, בדיקות ו־PR.

יש להעדיף רכיב משותף קיים או Adapter ממוקד לדף על פני שכפול מנגנון, Worker, Webhook, טופס או לוגיקת מדידה.

## אבטחת הוראות העבודה

`AGENTS.md` ותיקיית `skills/` הם קבצי עבודה פנימיים. חובה להשאיר אותם מוחרגים מפריסת FTP לאתר הציבורי.
